# Дүгнэлт

### Нэвтрэлтийн гол шалтгаан: Request Tracker (RT) вэбд анхдагч итгэмжлэл (root/password) хэвээр байснаар довтлогч RT-д нэвтэрч, хэрэглэгч lnorgaard-ын бүртгэл дээр хадгалсан тодорхой текст нууц үг (Welcome2023!)-ийг олж авч SSH-р хэрэглэгчийн эрхтэй орсон.

### Дотоод мэдээллийн алдагдал: Хэрэглэгчийн гэрт KeePassDumpFull.dmp (санах ойн dump) ба passcodes.kdbx (KeePass өгөгдлийн сан) ил хадгалагдаж байсан. KeePass-ийн CVE-2023-32784 маягийн сул тал/аргачлалаар dump-аас мастер нууц үгийн тэмдэгтүүдийг сэргээж, улмаар .kdbx-ийг нээж PuTTY хэлбэрийн SSH хувийн түлхүүр-ийг гарган авсан.

# Кибер тэсвэртэй байдлын зөвлөмж

### Анхдагч нууц үг бүрмөсөн хориглох: RT зэрэг системүүд дээр root/password гэх мэт default-г шуудайгаар устгах, анхны суулгалтын үе шатанд хүчтэй нууц үг + MFA заавал тавьдаг журам тогтоох.

### Санах ойн dump-ыг цэгцлэх: .dmp зэрэг мэдрэмтгий файлуудыг орон нутгийн дискт ил задгай үлдээхгүй, хэрэгцээ дуусмагц устгах; дамжуулах бол шифрлэсэн архив ашиглах.
